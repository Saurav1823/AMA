
# Difference between SQL and NoSQL

- SQL databases are relational and store data in tables (rows and columns).  
- They use fixed schema (predefined structure).  
- NoSQL databases are non-relational.  
- They use flexible schema and different data models like document or key-value.  

---

# Atomicity in SQL

- Atomicity means all operations in a transaction happen completely or not at all.  
- If one operation fails, the entire transaction is cancelled.  

---

# Man Command

- `man` command shows the manual/help page of a command in Linux or Unix systems.  

---

# ACID Properties

- Atomicity → All or nothing execution  
- Consistency → Data remains correct and valid  
- Isolation → Transactions run independently  
- Durability → Data is permanently saved after commit  

---

# What is Transaction

- A transaction is a group of database operations executed as a single unit.  

---

# Difference between LIMIT and OFFSET

- LIMIT decides how many rows to display.  
- OFFSET decides from which row to start displaying.  

---

# Set in Python

- A set is a collection of unique elements.  
- Duplicate values are not allowed.  

---

# Wildcards in SQL

- Wildcards are special characters used with LIKE for pattern matching.  
- `%` represents any number of characters.  
- `_` represents exactly one character.  

---

# Use of CAP Theorem

- CAP theorem states that a distributed system can provide only two out of three:  
  Consistency, Availability, and Partition Tolerance.  

---

# Inheritance in Python

- Inheritance means one class can use properties and methods of another class.  
- It helps in code reuse.  

---

# Membership Operators in Python

- `in` checks if a value exists in a sequence.  
- `not in` checks if a value does not exist.  

---

# Difference between Primary Key and Unique

- Primary key does not allow duplicate and NULL values.  
- Unique key does not allow duplicate values but can allow NULL.  
- Only one primary key per table, but multiple unique keys allowed.  

---

# DISTINCT in SQL

- DISTINCT removes duplicate records from the result.  

---

# Time Complexity in Python

- Time complexity tells how execution time increases with input size.  
- Common examples: O(1), O(n), O(log n).  

---

# When to Use LIKE

- LIKE is used to search for patterns in text columns.
